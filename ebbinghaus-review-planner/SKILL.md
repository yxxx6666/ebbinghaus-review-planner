# 艾宾浩斯复习规划大师

Version: v1.2.0

## 1. 目标

把用户明确要学习或长期掌握的资料，转成一套可立即执行的学习系统：

1. 知识地图
2. 分级知识点
3. 主动回忆题
4. 具体日期的复习计划
5. 根据掌握反馈自动调整的每日任务
6. 经过审核的 A4 可打印学习手册（PDF）和可编辑版本（DOCX）

不要把“艾宾浩斯曲线”当成唯一固定时间表。使用间隔重复、主动回忆、知识点分级、错误反馈和应用检测共同规划。最终交付不能停留在聊天文本：默认把内容制作成美观、清晰、可打印的 A4 文件，并在逐页审核通过后交付。

## 2. 触发边界

仅在用户明确表达以下意图时执行：学习、记忆、复习、备考、培训、内化、长期掌握、生成复习表、制作闪卡、安排每日学习任务或导出复习数据。

不要仅因用户上传 PDF、Word、PPT、书籍、笔记或课程资料就触发。

以下请求不要使用本 Skill，除非用户同时要求学习或复习系统：

- 只总结、概括或提取资料
- 只翻译、润色或改写
- 只分析观点或回答单个问题
- 只转换文件格式
- 只制作演示文稿、文档或图片

## 3. 核心规则

1. 先忠实理解资料，再拆知识点；不得补写资料中没有的事实。
2. 复习以闭卷回忆、做题、复述、讲解和应用为主，不把“再看一遍”当成完成。
3. A/B 级知识点必须至少有一道可检测的主动回忆题。
4. 不同知识类型匹配不同题型；读取 `references/knowledge-card-types.md`。
5. 默认先用结构化 Markdown 作为内容底稿，再制作 A4 成品；不默认绑定 Notion、Excel、Anki 或日历。
6. 默认最终交付 A4 竖版 PDF 和可编辑 DOCX。仅当用户明确要求纯文本/特定平台格式，或当前环境确实无法创建文件时，才不生成文件。
7. 日期使用用户时区和具体年月日；无法确认日期时才使用 Day 0、Day 1。
8. 长资料先完成可执行的首批计划和第一册，不制造超大表格或单个失控文件。
9. 对预计学习时长只做规划估算，并明确是估算值。
10. 原文依据尽量可定位；无法精确定位时给出章节、段落或关键原句。
11. 题目区与答案区必须分开，避免答案紧挨问题破坏主动回忆。
12. 文件生成后必须经过内容审核和逐页视觉审核；未审核通过不得声称完成。

## 4. 执行流程

### 第一步：识别任务

判断：

- 资料类型：考试、课程、书籍、工作、企业培训、SOP、产品知识、销售话术、论文、科普或其他
- 学习目标：考试记忆、工作应用、课程学习、企业培训、兴趣自学、内容创作
- 学习开始日：优先使用用户明确提供的日期，否则使用当前日期
- 截止日或考试日：有则记录，没有则不虚构
- 每日预算：优先使用用户提供的时长；未提供时按 30 分钟规划
- 输出平台：默认生成 A4 PDF + DOCX；用户明确要求时再追加 Notion、Excel、Anki、日历或纯文本

如果学习目标未说明，采用“通用学习模式”。

### 第二步：选择输出规模

按下列规则选择，不必先询问：

- **快速版**：用户说“简单、快速、先给我能执行的版本”，或材料不超过约 1500 个中文字符 / 1000 个英文单词。
- **标准版**：默认模式，适合一个章节、一篇中长文或一组普通课程笔记。
- **完整版**：用户明确要求完整题库、完整数据库、逐章覆盖或全量导出。

材料特别长时，即使用户要求完整版，也先输出总体知识地图、首批 A 级知识点、首个 7 天计划和拆分批次，再继续扩展。

读取 `references/output-templates.md` 获取各模式的字段和数量上限。

### 第三步：建立知识地图

拆出：一级主题、二级知识点、关键概念、定义、步骤、公式或数字、因果关系、对比关系、易错点、应用场景和可出题点。

每个知识点标记：

- 重要程度：A 必须掌握 / B 重点理解 / C 了解即可 / D 可跳过或低频
- 难度：1 简单 / 2 中等 / 3 较难 / 4 很难
- 类型：定义、概念、流程、步骤、公式、数字、对比、案例、话术、应用、理解或判断

优先把知识点切成一次能在 30 秒到 3 分钟内回忆或检测的小单元。避免一张卡塞入多个独立结论。

### 第四步：生成主动回忆题

按知识类型选择题型：问答、填空、判断、选择、场景、流程复述、对比、一句话解释、讲给别人听或实操任务。

参考答案必须：

- 忠于原资料
- 包含必要关键词
- 简短到可以复习
- 写清检测标准

如果资料不足以支持答案，写“资料中未明确说明”，不要推测。

### 第五步：安排复习

根据学习目标选择：标准版、忙碌简化版、考试冲刺版或兴趣轻量版。读取 `references/review-schedules.md`。

日期规则：

1. 优先使用用户给出的实际学习日期。
2. 未给出时，以当前日期为 Day 0。
3. 所有可计算任务写成 `YYYY-MM-DD`。
4. R0 与当天晚间任务间隔不足 2 小时时合并。
5. 同一天出现多个阶段时合并为一个任务，按 A 级、薄弱点、B 级、C 级排序。
6. 有考试或截止日时，不安排截止日后的冲刺任务；考前 1 天保留最后一次轻量复习。
7. 用户未要求长期保留时，考试后的 30/60/90 天任务不自动生成。
8. 每日任务超过预算时，先保留 A 级和薄弱点，再拆分或顺延 B/C 级。
9. 学习开始日在未来时，不安排开始日前任务。

### 第六步：应用自适应评分

复习后使用 0-4 分：

- 0：完全不会
- 1：模糊，需要明显提示
- 2：能答出主要内容，但不稳定
- 3：能独立准确回答或讲清楚
- 4：能在新场景中应用、比较或举例

计算下一次复习间隔时，先取：

`基础间隔 = max(1, 距上次复习的天数)`

再按评分调整：

| 评分 | 下一次间隔 |
|---|---|
| 0 | 1 天，并安排当天重新学习 |
| 1 | `max(1, round(基础间隔 × 0.5))` 天 |
| 2 | `max(1, round(基础间隔 × 1.2))` 天 |
| 3 | `min(90, max(2, round(基础间隔 × 2)))` 天 |
| 4 | `min(90, max(3, round(基础间隔 × 3)))` 天 |

附加规则：

- 连续 2 次评分 0 或 1：标记“薄弱点”，退回重新学习，并生成一条纠错任务。
- 连续 3 次评分 3 或 4：进入“长期维护”，后续只在 30、60 或 90 天级别抽查。
- 最短间隔 1 天，最长间隔 90 天。
- 用户只给文字反馈时映射：不会=0，模糊=1，会=2，能讲出来=3，能应用=4。

### 第七步：内容审核

在排版前先完成内容审核：

1. 核对知识点、题目和答案是否忠于资料。
2. 核对 A/B 级知识点是否被题目覆盖。
3. 核对题号、知识点编号、答案编号和原文位置是否一一对应。
4. 核对具体日期、截止日、复习间隔和每日预算。
5. 删除重复题、空泛题、无法评分的题和资料不支持的答案。

未通过内容审核时先修正底稿，不进入排版。

### 第八步：制作 A4 可打印成品

读取 `references/a4-print-workflow.md`，默认采用 DOCX 作为可编辑母版，再转换为 PDF。成品至少包含：

1. 封面与使用说明
2. 学习总览和知识地图
3. 核心知识点
4. 主动回忆练习区（不显示答案）
5. 复习日历与每日任务
6. 自评分与薄弱点记录区
7. 独立答案与检测标准区

快速版也必须形成简洁的可打印小册，而不是只把聊天表格原样塞进文件。

### 第九步：渲染与逐页审核

必须执行“生成 -> 渲染为页面图片 -> 逐页检查 -> 修正 -> 重新渲染”的闭环：

1. 检查 DOCX 每一页。
2. 转换为最终 PDF。
3. 检查 PDF 每一页，并运行 A4 页面尺寸预检脚本。
4. 发现文字裁切、重叠、断表、乱码、空白页、孤立标题、页码错误或答案编号错位时，必须修复并重新审核。
5. 只有全部页面通过，才交付最终文件。

### 第十步：交付

默认交付：

- `资料名称-学习复习手册.pdf`：打印版
- `资料名称-学习复习手册.docx`：可编辑版

可同时给出一句简短说明：覆盖范围、页数、学习开始日、审核状态。不要把内部逐项检查过程全部展示给用户。

## 5. 原文定位规则

- PDF：页码；能定位时加章节或小标题
- PPT：幻灯片编号
- 视频或音频逐字稿：时间戳
- 书籍：章节、小节或页码
- 网页或长文：标题、小标题和段落
- 用户粘贴文本：段落编号或关键原句
- 无法定位：写“位置未提供”，并附最短必要原句

## 6. 学习时长估算

只把时长作为任务规划启发，不声称是科学测量。可使用：

1. 估算初读时间。
2. 复杂公式、法律、医学、技术或高密度材料乘以 1.5-2。
3. 主动回忆和练习时间单独计算。
4. 最终写成范围，例如“约 35-50 分钟”。

如果资料内容不可见或长度未知，不给出精确分钟数，改写为“需根据完整材料长度估算”。

## 7. 输出模式

### 快速版

必须制作成简洁的 A4 打印小册，并包含：

1. 学习目标与资料判断
2. 5-12 个最重要知识点
3. 每个 A 级知识点至少一道主动回忆题
4. 从学习日开始的 7 天任务
5. 掌握评分说明

### 标准版

制作成完整 A4 学习手册，包含：学习总览、知识地图、核心知识点、主动回忆练习、复习时间表、每日任务、自评记录、独立答案区和可选导出说明。遵守模板中的数量上限。

### 完整版

覆盖全部重要章节和 A/B 级知识点，可追加完整题库、数据库字段或导入文件。超长资料必须分册或分批编号，保持知识点编号稳定；每册都要独立通过 A4 打印与逐页审核。

## 8. 平台适配

默认用 Markdown 生成内容底稿，并以 A4 PDF + DOCX 作为最终成品。用户还要求其他平台时，读取：

- `references/output-templates.md`
- `references/platform-adapters.md`

支持 Notion、Excel/CSV、Anki、日历、手机备忘录和每日打卡版；这些属于附加导出，不替代默认 A4 成品。

## 9. 资料过长时

先交付：

1. 总体知识地图
2. 章节优先级
3. 第一批 A 级知识点
4. 第一批主动回忆题
5. 首个 7 天复习计划
6. 后续批次拆分表

不要只说“内容太长”。不要把未处理的内容假装成已完整覆盖。

## 10. 禁止事项

- 禁止仅输出摘要后宣称完成复习规划。
- 禁止因用户上传学习资料而自动假定其需要复习计划。
- 禁止把所有知识点做成同一种问答卡。
- 禁止编造考点、答案、数字、案例或原文位置。
- 禁止把“看一遍”作为唯一复习动作。
- 禁止默认生成超宽、超长数据库。
- 禁止在有截止日时继续机械生成无关的远期任务。
- 禁止给不可见或未知长度的资料编造精确学习时长。
- 禁止只输出聊天文本或 Markdown 就声称已完成最终交付。
- 禁止把问题和参考答案紧挨排在同一表格中作为打印练习页。
- 禁止未经逐页渲染检查就交付 DOCX 或 PDF。
- 禁止用小于 9 磅的正文或缩小到难以打印阅读来解决溢出。
- 禁止出现被裁切文字、表格出界、重叠、乱码、错误页码或答案编号错位。
