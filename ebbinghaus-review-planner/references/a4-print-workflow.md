## 1. 默认交付

除非用户明确要求纯文本或其他格式，最终交付两个文件：

- A4 PDF：直接打印和阅读。
- A4 DOCX：后续编辑和填写。

不要只提供 Markdown、聊天表格或截图。

## 2. 内容架构

采用“先学、后练、再核对”的顺序：

1. 封面。
2. 使用说明和 0-4 分评分方法。
3. 学习总览。
4. 知识地图。
5. 核心知识点。
6. 主动回忆练习题。
7. 复习日历和每日任务。
8. 薄弱点与复习记录。
9. 参考答案和检测标准。

问题和答案必须分区。不要把参考答案放在题目正下方或同一张打印表格中。

## 3. A4 版式规范

### 页面

- 纸张：A4，210 x 297 mm。
- 默认方向：竖版。只有用户明确要求或无法合理重排的极宽数据才使用横版页。
- 页边距：建议上下 16-20 mm，左右 18-22 mm；保证家用打印机安全边界。
- 页眉页脚：保持简洁，页脚包含页码；可加入资料简称。

### 字体与层级

- 使用当前环境可稳定显示简体中文的字体，不捆绑或分享字体文件。
- 正文建议 10.5-11.5 磅；任何正文不得小于 9 磅。
- 一级标题建议 18-22 磅，二级标题 14-16 磅，三级标题 11.5-13 磅。
- 行距建议 1.25-1.45；段后留白保持一致。
- 不使用花体、超细字或低对比灰字。

### 色彩

- 白色或极浅背景，深色正文。
- 只使用 1 个主强调色和少量浅色块。
- 所有信息在黑白打印时仍可理解；不能只靠颜色表达等级或状态。
- 重要程度同时用 A/B/C/D 文本、符号或边框区分。

### 表格

- A4 竖版尽量不超过 5-6 列。
- 宽表优先拆表、转成纵向卡片或每题一块，而不是无限缩小字号。
- 跨页表格重复表头。
- 避免一行被拆到两页；无法避免时确保题号和内容仍可识别。
- 单元格内不堆大段文字；答案长时使用编号列表。

### 练习可写性

- 选择题保留清晰选项间距。
- 问答题留出横线、空白框或要点区。
- 每题提供自评分 0/1/2/3/4 和薄弱点勾选框。
- 每日任务提供“完成、评分、问题、下一步”记录位。

## 4. DOCX 制作

文本型学习手册优先用 DOCX 作为母版。创建时：

1. 明确设置 A4 页面和页边距。
2. 使用样式管理标题、正文、表格和提示框，不靠手工逐段调整。
3. 使用分页符控制章节，不用大量空行顶页。
4. 标题不得孤立在页尾；至少与后续两行内容同页。
5. 答案章节另起页。
6. 表格跨页时重复标题行。
7. 页码从正文或封面后按设计统一处理。

在 OpenAI 容器环境中，遵循 DOCX capability 的 render-and-verify 工作流。可使用：

```bash
python /home/oai/skills/docx/render_docx.py input.docx --output_dir out_docx --emit_pdf
```

渲染图只用于内部审核，不作为默认交付物。

## 5. PDF 转换

优先从审核后的 DOCX 转换为 PDF，不要为长文本手工拼接固定坐标 PDF。

在 OpenAI 容器环境中可使用 LibreOffice 转换脚本或 DOCX 渲染器生成 PDF。转换后必须再次审核 PDF，因为分页和字体可能变化。

## 6. 渲染与逐页审核

### DOCX 审核

1. 渲染每一页为 PNG。
2. 逐页查看 100% 视图。
3. 检查页边距、标题、表格、中文字体、页码和书写区域。
4. 修正后重新渲染全部页面。

### PDF 审核

在 OpenAI 容器环境中可使用：

```bash
python /home/oai/skills/pdfs/scripts/render_pdf.py final.pdf --out_dir out_pdf --dpi 200
```

逐页检查：

- 文字是否被裁切或重叠。
- 是否有缺字、乱码、黑块。
- 表格是否出界、断裂或字号过小。
- 页码是否连续。
- 问题与答案编号是否对应。
- 是否出现意外空白页。
- 黑白打印是否可读。

任何一项失败，都要回到 DOCX 修正并重新生成 PDF。

## 7. 自动预检

运行 `scripts/check_a4_pdf.py` 检查：

- 所有页面是否接近 A4 尺寸。
- 页面方向是否符合默认竖版。
- PDF 是否存在空白或近空白页。
- 文件是否可正常打开。

自动预检不能替代视觉审核。脚本通过也必须逐页查看。

## 8. 修复优先级

遇到页面拥挤时按以下顺序修复：

1. 精简重复文字。
2. 拆分宽表和长表。
3. 调整列宽、段前段后和页边距。
4. 增加页面或分册。
5. 最后才轻微缩小字号，但正文不得小于 9 磅。

不要通过极小字体、极窄边距或强行压缩行距解决溢出。

## 9. 文件命名与交付

使用可识别文件名：

- `资料名称-学习复习手册.pdf`
- `资料名称-学习复习手册.docx`

文件名过长时保留核心资料名。最终只交付成品，删除或隐藏渲染 PNG、临时 PDF、日志和中间文件。
