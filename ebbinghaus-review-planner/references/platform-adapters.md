# 平台适配规则

## 1. 默认平台中立

默认输出 Markdown 和纯文本。用户不使用专业软件时，也必须能直接复制执行。

## 2. Notion

仅在用户明确要求时输出字段。优先采用“一条知识点一行 + 下次复习日期”的纵向结构，避免横向堆叠大量复习日期字段。

建议状态：未学习、待复习、薄弱点、已掌握、长期维护。

## 3. Excel / Google Sheets

输出 UTF-8 CSV 字段和可直接粘贴的表格。日期统一 `YYYY-MM-DD`，评分为 0-4 的数字，间隔为整数天。

## 4. Anki

Front 只放一个可检测问题。Back 放简短答案、必要解释和原文位置。不要把整段材料塞进一张卡。

## 5. 日历

仅在用户要求日历提醒时输出。将同一天任务合并为一个事件，标题包含资料名称和阶段，描述包含题号、动作和预计用时。

## 6. 手机备忘录或聊天工具

优先输出当天任务和最近 7 天计划，减少数据库字段。每条任务采用“动作 + 题号 + 用时 + 评分”结构。

## 7. A4 打印版

A4 PDF + DOCX 是默认最终成品，不是可选附录。具体执行读取 `references/a4-print-workflow.md`。

使用窄表格：日期、任务、题号、自评分、薄弱点、下一步。避免依赖交互字段。题目和答案分成不同章节，题目页留出书写空间。颜色必须高对比并兼容黑白打印。
